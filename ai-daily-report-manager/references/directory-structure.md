# 标准目录结构

## 概览
本文档定义了AI日报项目的标准目录结构，用于确保文件归类的合理性。

## 核心内容

### 目录树结构
```
daily-reports/
├── daily/              # 日报文件目录
├── review/             # 复盘报告目录
├── template/           # 模板文件目录
├── archive/            # 归档文件目录
├── temp/               # 临时文件目录
└── backup/             # 备份文件目录
```

### 目录用途说明

#### `daily/` - 日报文件目录
- **用途**：存放所有AI日报文件
- **文件类型**：`ai-daily-YYYY-MM-DD.md`
- **命名规范**：见 `naming-convention.md`
- **文件归类规则**：
  - 所有日报文件必须存放在此目录下
  - 按日期顺序排列
  - 禁止在此目录下创建子目录
- **示例**：
  ```
  daily/
  ├── ai-daily-2024-01-01.md
  ├── ai-daily-2024-01-02.md
  └── ai-daily-2024-01-03.md
  ```

#### `review/` - 复盘报告目录
- **用途**：存放文件管理复盘报告
- **文件类型**：`review-YYYY-MM-DD.md`
- **命名规范**：见 `naming-convention.md`
- **文件归类规则**：
  - 所有复盘报告必须存放在此目录下
  - 按复盘日期排序
  - 禁止在此目录下创建子目录
- **示例**：
  ```
  review/
  ├── review-2024-01-07.md
  └── review-2024-02-01.md
  ```

#### `template/` - 模板文件目录
- **用途**：存放日报和复盘的模板文件
- **文件类型**：`template-*.md`
- **命名规范**：见 `naming-convention.md`
- **文件归类规则**：
  - 所有模板文件必须存放在此目录下
  - 禁止在此目录下创建子目录
- **示例**：
  ```
  template/
  ├── template-daily.md
  └── template-review.md
  ```

#### `archive/` - 归档文件目录
- **用途**：存放已归档的历史文件
- **文件类型**：`archive-YYYY-MM-DD.md`
- **命名规范**：见 `naming-convention.md`
- **文件归类规则**：
  - 超过保留期限的文件移动到此目录
  - 按年份创建子目录（可选）
- **示例**：
  ```
  archive/
  ├── 2023/
  │   ├── archive-2023-12-01.md
  │   └── archive-2023-12-31.md
  └── 2024/
      └── archive-2024-01-01.md
  ```

#### `temp/` - 临时文件目录
- **用途**：存放临时创建的文件
- **文件类型**：任意格式
- **命名规范**：无特殊要求
- **文件归类规则**：
  - 临时文件应尽快整理到正确目录
  - 禁止长期存放文件（建议不超过1周）
  - 整理时应清空此目录
- **示例**：
  ```
  temp/
  ├── 临时草稿.md
  └── 未命名文件.txt
  ```

#### `backup/` - 备份文件目录
- **用途**：存放文件整理时的备份
- **文件类型**：`.bak_*` 扩展名的备份文件
- **命名规范**：`<原文件名>.bak_<timestamp>`
- **文件归类规则**：
  - 仅由脚本自动创建和管理
  - 禁止手动创建或修改备份文件
  - 定期清理过期备份（如保留30天）
- **示例**：
  ```
  backup/
  ├── ai-daily-2024-01-01.md.bak_20240101_120000
  └── ai-daily-2024-01-02.md.bak_20240102_120000
  ```

## 文件归类规则

### 规则1：按文件类型归类
- 日报文件 → `daily/`
- 复盘文件 → `review/`
- 模板文件 → `template/`
- 归档文件 → `archive/`

### 规则2：按文件名前缀识别
- 以 `ai-daily-` 开头 → `daily/`
- 以 `review-` 开头 → `review/`
- 以 `template-` 开头 → `template/`
- 以 `archive-` 开头 → `archive/`

### 规则3：临时文件处理
- 存放在 `temp/` 目录下的文件，应在整理时移动到正确目录
- 脚本会自动识别 `temp/` 目录下的日报文件并移动到 `daily/`

## 约束与注意事项

1. **禁止创建非标准目录**：除上述标准目录外，禁止创建其他目录
2. **禁止深层嵌套**：标准目录下禁止创建子目录（`archive/` 除外，可按年份分层）
3. **禁止跨目录存放**：同一类型的文件只能存放在对应的目录下
4. **禁止混存文件**：不同类型的文件不能混存在同一目录下

## 验证方法

扫描脚本（`scan_files.py`）使用以下规则验证目录结构：

1. **目录存在性检查**：检查是否存在所有标准目录
2. **文件位置验证**：检查文件是否存放在正确的目录下
3. **非标准目录检测**：检测是否存在不符合规范的目录

## 示例

### 正确的目录结构
```
daily-reports/
├── daily/
│   ├── ai-daily-2024-01-01.md
│   ├── ai-daily-2024-01-02.md
│   └── ai-daily-2024-01-03.md
├── review/
│   └── review-2024-01-07.md
├── template/
│   ├── template-daily.md
│   └── template-review.md
├── archive/
│   └── 2023/
│       └── archive-2023-12-31.md
├── temp/
│   └── (空)
└── backup/
    └── (空)
```

### 错误的目录结构（需要整理）
```
daily-reports/
├── daily/
│   └── ai-daily-2024-01-01.md
├── temp/
│   └── ai-daily-2024-01-02.md      # 应移动到 daily/
├── old_files/                      # 非标准目录，应删除或归档
│   └── ai-daily-2023-12-01.md
└── review-2024-01-07.md            # 应移动到 review/ 目录
```
