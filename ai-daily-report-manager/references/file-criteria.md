# 文件完整性检查标准

## 概览
本文档定义了AI日报文件的完整性检查标准，用于评估文件的质量和规范性。

## 核心内容

### 日报文件完整性标准

#### 必填字段清单
日报文件（`ai-daily-YYYY-MM-DD.md`）必须包含以下字段：

1. **日期字段**
   - 格式：`## YYYY-MM-DD`
   - 位置：文件开头（第一行或第二行）
   - 示例：`## 2024-01-01`
   - 验证规则：
     - 必须存在
     - 日期格式必须为 `YYYY-MM-DD`
     - 日期必须与文件名中的日期一致

2. **内容概要**
   - 格式：`### 概要` 或 `## 概要`
   - 位置：日期字段之后
   - 内容要求：
     - 简要描述当日工作的主要成果
     - 字数要求：50-200字
   - 验证规则：
     - 必须存在
     - 不能为空
     - 必须包含实质性内容

3. **详细内容**
   - 格式：`### 详细内容` 或 `## 详细内容`
   - 位置：概要之后
   - 内容要求：
     - 详细描述当日工作内容
     - 包含至少2个以上的具体事项
   - 验证规则：
     - 必须存在
     - 不能为空
     - 包含具体的工作描述

4. **遇到的问题**
   - 格式：`### 遇到的问题` 或 `## 遇到的问题`
   - 位置：详细内容之后
   - 内容要求：
     - 记录当日工作中遇到的问题
     - 如果没有问题，应明确标注"无"
   - 验证规则：
     - 必须存在
     - 如果有内容，必须描述具体问题

5. **后续计划**
   - 格式：`### 后续计划` 或 `## 后续计划`
   - 位置：遇到的问题之后
   - 内容要求：
     - 记录后续的工作计划
     - 包含至少1个具体的计划事项
   - 验证规则：
     - 必须存在
     - 不能为空
     - 包含可执行的计划

#### 可选字段
- **备注**：记录其他需要说明的事项
- **参考资料**：记录相关的参考文档或链接

### 复盘文件完整性标准

#### 必填字段清单
复盘文件（`review-YYYY-MM-DD.md`）必须包含以下字段：

1. **复盘日期**
   - 格式：`## YYYY-MM-DD 复盘报告`
   - 位置：文件开头
   - 示例：`## 2024-01-07 复盘报告`
   - 验证规则：
     - 必须存在
     - 必须包含"复盘报告"字样

2. **复盘周期**
   - 格式：`### 复盘周期`
   - 位置：日期之后
   - 内容要求：
     - 明确复盘的时间范围（如：2024-01-01 至 2024-01-07）
   - 验证规则：
     - 必须存在
     - 必须包含具体的日期范围

3. **问题分析**
   - 格式：`### 问题分析`
   - 位置：复盘周期之后
   - 内容要求：
     - 列出周期内发现的主要问题
     - 每个问题应包含问题描述和原因分析
   - 验证规则：
     - 必须存在
     - 至少包含1个问题分析

4. **改进方案**
   - 格式：`### 改进方案`
   - 位置：问题分析之后
   - 内容要求：
     - 针对每个问题提出具体的改进方案
     - 改进方案应具有可执行性
   - 验证规则：
     - 必须存在
     - 与问题分析中的问题一一对应

5. **效果评估**
   - 格式：`### 效果评估`
   - 位置：改进方案之后
   - 内容要求：
     - 评估改进方案的实施效果
     - 提供量化的评估数据（如：文件规范率从60%提升到90%）
   - 验证规则：
     - 必须存在
     - 必须包含评估结果

6. **后续计划**
   - 格式：`### 后续计划`
   - 位置：效果评估之后
   - 内容要求：
     - 明确下一步的优化方向和具体措施
   - 验证规则：
     - 必须存在
     - 不能为空

## 质量评分规则

### 日报文件评分
总分：100分

1. **字段完整性**（40分）
   - 所有必填字段完整：40分
   - 缺少1个字段：-10分
   - 缺少2个字段：-20分
   - 缺少3个及以上字段：-30分

2. **内容质量**（30分）
   - 内容详实、描述清晰：30分
   - 内容基本完整、描述清晰：20分
   - 内容简单、描述粗糙：10分
   - 内容空洞、描述不清晰：0分

3. **格式规范性**（20分）
   - 符合Markdown格式规范：20分
   - 基本符合Markdown格式规范：10分
   - 格式混乱：0分

4. **日期一致性**（10分）
   - 文件名与内容日期完全一致：10分
   - 日期不一致：0分

### 复盘文件评分
总分：100分

1. **字段完整性**（40分）
   - 所有必填字段完整：40分
   - 缺少1个字段：-10分
   - 缺少2个字段：-20分
   - 缺少3个及以上字段：-30分

2. **问题分析深度**（30分）
   - 问题分析深入、原因明确：30分
   - 问题分析基本到位：20分
   - 问题分析浅显：10分
   - 问题分析缺失：0分

3. **改进方案可行性**（20分）
   - 改进方案具体、可执行：20分
   - 改进方案基本可行：10分
   - 改进方案模糊、难以执行：0分

4. **效果评估客观性**（10分）
   - 评估数据客观、量化：10分
   - 评估基本客观：5分
   - 评估主观、无数据支持：0分

## 验证方法

### 自动化验证规则
评估脚本（`evaluate_structure.py`）使用以下规则验证文件完整性：

1. **必填字段检测**
   - 搜索文件内容中的必填字段标题
   - 检查字段是否存在

2. **日期一致性检查**
   - 提取文件名中的日期
   - 提取文件内容中的日期
   - 比较两者是否一致

3. **内容空值检测**
   - 检查必填字段下是否为空
   - 检查是否仅有占位符（如"待补充"、"TODO"）

4. **格式规范性检查**
   - 检查是否使用Markdown格式
   - 检查标题层级是否合理

### 人工验证规则
智能体在生成复盘报告时应进行以下人工验证：

1. **内容质量评估**
   - 检查内容是否充实
   - 检查描述是否清晰
   - 检查是否有实质性的信息

2. **逻辑一致性检查**
   - 检查问题分析与改进方案是否对应
   - 检查效果评估是否与改进方案一致
   - 检查后续计划是否合理

3. **可执行性评估**
   - 检查改进方案是否具体
   - 检查后续计划是否可执行
   - 检查是否需要额外的资源或支持

## 示例

### 完整的日报文件（满分）
```markdown
## 2024-01-01

### 概要
今日完成了AI日报管理系统的初步设计，包括文件命名规范、目录结构定义和扫描脚本开发。

### 详细内容
1. 完成了文件命名规范的定义，明确了日报、复盘、模板文件的命名规则
2. 设计了标准目录结构，包含daily、review、template、archive、temp、backup六个目录
3. 开发了文件扫描脚本，能够识别命名不规范、位置错误、重复版本的文件

### 遇到的问题
无

### 后续计划
1. 完成文件整理脚本的开发
2. 完成目录结构评估脚本的开发
3. 进行第一次文件整理测试
```

### 不完整的日报文件（低分）
```markdown
## 2024-01-01

### 概要
TODO

### 详细内容
工作内容待补充

### 后续计划
待定
```

### 完整的复盘文件（满分）
```markdown
## 2024-01-07 复盘报告

### 复盘周期
2024-01-01 至 2024-01-07

### 问题分析
1. 文件命名不规范：部分文件使用20240101.txt格式，不符合ai-daily-YYYY-MM-DD.md规范
2. 文件位置错误：日报文件散落在根目录和temp目录下，未统一归位到daily/目录
3. 重复文件存在：同一文件在多个位置重复存放，造成存储浪费

### 改进方案
1. 扫描识别不规范文件，生成扫描报告
2. 执行文件整理操作，重命名、移动、合并文件
3. 建立自动化检查机制，定期评估文件系统状态

### 效果评估
- 文件规范率从60%提升到95%
- 文件位置正确率达到100%
- 消除了所有重复文件，节省存储空间约5MB

### 后续计划
1. 将复盘结果沉淀为技能文档，作为后续整理的参考
2. 建立定期复盘机制，每月进行一次文件系统评估
3. 持续优化文件管理流程，提高自动化程度
```
